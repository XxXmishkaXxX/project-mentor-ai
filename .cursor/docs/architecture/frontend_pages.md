## Дизайн-система

### Общая тема

Тёмная «космическая» тема. Все страницы используют единый анимированный фон.

### Анимированный фон (Cosmic Background)

На всех страницах приложения присутствует анимированный фон:

- **Базовый градиент**: радиальный или линейный, от глубокого тёмно-синего (`#070B1A`) через тёмно-фиолетовый (`#0D1033`) к почти чёрному (`#050510`)
- **Частицы (точки)**: мелкие светящиеся точки (1–3px), белые и фиолетовые с разной прозрачностью (0.3–0.8), медленно дрейфующие по экрану
- **Линии (connections)**: между близлежащими точками рисуются полупрозрачные линии (opacity 0.1–0.3), создавая эффект сетки/констелляции
- **Движение**: точки плавно перемещаются со скоростью ~0.5–1px/frame; линии динамически появляются/исчезают при сближении/удалении точек
- **Реализация**: Canvas-элемент на `position: fixed; z-index: 0` за всем контентом. Можно использовать библиотеку `particles.js` / `tsparticles` или кастомный Canvas-рендер
- **Производительность**: `requestAnimationFrame`, ~60–100 частиц максимум, фон не должен влиять на производительность UI

### Типографика

- **Основной шрифт**: `Inter` (подключить через Google Fonts), fallback `system-ui, -apple-system, sans-serif`
- **Размеры**: body `14px`, h1 `24px`, h2 `20px`, h3 `16px`, small `12px`
- **Line-height**: 1.5 для текста, 1.3 для заголовков
- **Font-weight**: normal `400`, medium `500`, semibold `600`, bold `700`

### Скругления (border-radius)

| Элемент | Радиус |
|---------|--------|
| Кнопки | `8px` |
| Карточки | `12px` |
| Поля ввода | `8px` |
| Пузыри сообщений | `16px` (с `4px` на стороне «хвоста») |
| Модальные окна | `16px` |
| Аватары | `50%` (круг) |
| Чипы/бейджи | `6px` |

### Тени

- **Карточки**: `0 4px 24px rgba(0, 0, 0, 0.3)`
- **Модальные окна**: `0 8px 32px rgba(0, 0, 0, 0.5)`
- **Glow-эффект (кнопки)**: `0 0 20px rgba(108, 92, 231, 0.3)`

### Анимации и переходы

- Все hover/active: `transition: all 0.2s ease`
- Появление сообщений: `fadeInUp` (opacity 0→1, translateY 10px→0, 0.3s)
- Открытие модалок: `fadeIn` overlay + `slideUp` контент
- Пульсация «думает»: три точки с анимацией `pulse` поочерёдно

## Страница 1: Вход (LoginPage)

**Маршрут**: `/login`
**Auth**: публичный (если пользователь уже залогинен — редирект на `/chat`)

### Визуал

- Анимированный космический фон занимает весь экран
- По центру экрана (вертикально и горизонтально) — карточка формы входа
- **Карточка**: `width: 420px`, `background: rgba(13, 16, 51, 0.85)`, `backdrop-filter: blur(20px)`, `border: 1px solid rgba(255,255,255,0.08)`, `border-radius: 16px`, `padding: 40px`

### Содержимое карточки (сверху вниз)

1. **Логотип и название**: иконка щита с AI-символом (`🛡️` или SVG), текст «Проектный Ассистент», под ним подзаголовок `--color-text-secondary` «Ваш ИИ-ментор в проектной деятельности»
2. **Заголовок формы**: «Вход в систему», под ним «Добро пожаловать! Войдите в свой аккаунт»
3. **Поле Email**: label «Email», `<input type="email">`, placeholder «you@example.com»
4. **Поле Пароль**: label «Пароль», `<input type="password">`, иконка глаза для показа/скрытия
5. **Чекбокс «Запомнить меня»** + ссылка «Забыли пароль?» (справа, `--color-text-secondary`, на текущем этапе неактивная)
6. **Кнопка «Войти»**: на всю ширину, `background: --color-accent-green` (`#00D68F`), `color: #fff`, `font-weight: 600`, `height: 48px`, `border-radius: 8px`. Hover: яркость +10%. Disabled при загрузке (спиннер вместо текста)
7. **Ссылка на регистрацию**: «Нет аккаунта? Зарегистрироваться» — текст + ссылка `--color-accent-primary`

### Стили полей ввода

- `background: rgba(18, 20, 56, 0.6)`, `border: 1px solid rgba(255,255,255,0.08)`, `border-radius: 8px`, `height: 44px`, `padding: 0 16px`, `color: #fff`
- Focus: `border-color: --color-border-focus`, лёгкий glow
- Ошибка: `border-color: --color-accent-red`, текст ошибки красным под полем

### Получение данных (API)

- Кнопка «Войти» → `POST /api/auth/login` body: `{ email, password }`
- Успех (200): сервер ставит cookie `session_id`, ответ `UserResponse` → сохранить в `authStore.user` → `router.push('/chat')`
- 401: показать «Неверный email или пароль» красным текстом под формой
- 422: показать ошибки валидации под соответствующими полями

### Клиентская валидация

- Email: обязательно, формат email
- Пароль: обязательно, минимум 1 символ (серверная проверка длины)
- Кнопка disabled, пока поля пустые

## Страница 2: Регистрация (RegisterPage)

**Маршрут**: `/register`
**Auth**: публичный (если залогинен — редирект на `/chat`)

### Визуал

- Идентичная LoginPage по стилю: анимированный фон, центрированная карточка
- Карточка чуть выше из-за дополнительных полей

### Содержимое карточки

1. **Логотип и название** (как на LoginPage)
2. **Заголовок**: «Регистрация», подзаголовок «Создайте аккаунт для начала работы»
3. **Поле Username**: label «Имя пользователя», placeholder «ivan_petrov»
4. **Поле Email**: label «Email», placeholder «you@example.com»
5. **Поле Пароль**: label «Пароль», с иконкой глаза
6. **Поле Подтверждение пароля**: label «Подтвердите пароль»
7. **Кнопка «Зарегистрироваться»**: `--color-accent-green`, на всю ширину
8. **Ссылка**: «Уже есть аккаунт? Войти» → `/login`

### API

- `POST /api/auth/register` body: `{ username, email, password }`
- Успех (201) → редирект на `/login` с flash-сообщением «Регистрация успешна, войдите»
- 409 → «Пользователь с таким email или именем уже существует»
- 422 → ошибки валидации

### Клиентская валидация

- Username: 3–50 символов
- Email: формат email
- Пароль: минимум 8 символов
- Подтверждение: совпадает с паролем

## Страница 3: Чат (ChatPage) — основная

**Маршрут**: `/chat` (новый чат) или `/chat/:id` (существующий)
**Auth**: требуется (иначе → `/login`)

### Layout (3 зоны)

```
┌─────────────────┬──────────────────────────────────┐
│                 │  Header (title + actions)         │
│   Sidebar       ├──────────────────────────────────┤
│   (280px)       │                                  │
│                 │  Chat Window (messages)           │
│   Список        │  [scrollable]                    │
│   чатов         │                                  │
│                 ├──────────────────────────────────┤
│                 │  Chat Input (textarea + send)     │
├─────────────────┴──────────────────────────────────┘
```

Анимированный фон виден за прозрачными/полупрозрачными блоками.

### 3a: Sidebar (ChatSidebar)

- **Ширина**: `280px`, `background: rgba(13, 16, 51, 0.9)`, `backdrop-filter: blur(20px)`, `border-right: 1px solid var(--color-border)`
- **Фиксированный** по высоте экрана, `overflow-y: auto` для списка

**Содержимое (сверху вниз):**

1. **Кнопка «+ Новый чат»**: ширина 100% минус отступы, `height: 42px`, `background: var(--color-accent-gradient)`, белый текст, `border-radius: 8px`. Hover: glow-эффект
2. **Поле поиска**: `<input>` с иконкой лупы слева, placeholder «Поиск чатов...», `background: rgba(18, 20, 56, 0.6)`, фильтрация чатов на клиенте по title
3. **Список чатов**: группировка по датам:
   - **Сегодня** — label `--color-text-secondary`, `font-size: 12px`, `text-transform: uppercase`
   - **Вчера**
   - **Ранее** (>2 дней)
   - Каждый элемент: `padding: 10px 16px`, `border-radius: 8px`, текст title (truncate с `...`), дата `--color-text-muted`
   - Hover: `background: var(--color-bg-surface-hover)`
   - Активный: `background: var(--color-bg-surface)`, левая граница `3px solid var(--color-accent-primary)`
   - Кнопка удаления (иконка корзины) появляется при hover, клик → confirm-диалог
4. **Профиль пользователя** (внизу sidebar): аватар (первая буква username в круге `--color-accent-primary`), username, подпись «Студент» или role. Опционально кнопка logout

### 3b: Header чата

- `height: 60px`, `background: transparent` или `rgba(13, 16, 51, 0.5)`, `border-bottom: 1px solid var(--color-border)`
- Слева: **title чата** (h2, `--color-text-primary`). Если title null — «Новый чат»
- Справа: кнопка «Поделиться» (иконка), кнопка «...» (меню: переименовать, удалить)

### 3c: Область сообщений (ChatWindow) — пустой чат (Welcome)

Если в текущем чате нет сообщений:

- По центру экрана вертикально и горизонтально:
  1. **Иконка робота/ассистента**: большая (80px), стилизованная, с фиолетовым glow-эффектом (три точки как «чат» или аватар бота)
  2. **Заголовок**: «Привет! Я ваш проектный ассистент» — `font-size: 24px`, `font-weight: 600`
  3. **Подзаголовок**: «Задавайте вопросы по управлению проектами, Agile, Scrum, DevOps и другим практикам.» — `--color-text-secondary`, `font-size: 14px`
  4. **Метка**: «Примеры вопросов:» — `--color-text-secondary`, `font-size: 13px`
  5. **Карточки примеров** (2 колонки, 2 ряда):
     - «Как правильно составить ТЗ?»
     - «Какие этапы у Scrum?»
     - «Как распределить роли в команде?»
     - «Что такое Kanban?»
     - Каждая карточка: `background: rgba(18, 20, 56, 0.6)`, `border: 1px solid var(--color-border)`, `border-radius: 12px`, `padding: 14px 18px`, иконка-стрелка справа `--color-accent-primary`
     - Hover: `border-color: var(--color-accent-primary)`, `background: rgba(108, 92, 231, 0.1)`
     - Клик: если нет текущего чата → `createChat()`, затем `sendMessage(text)`

### 3c: Область сообщений — заполненный чат

- `overflow-y: auto`, `padding: 24px`, flex-колонка с `gap: 16px`
- Сообщения идут в хронологическом порядке
- Автоскролл вниз при новых сообщениях и при стриминге

**Сообщение пользователя:**
- Выравнивание вправо, `max-width: 70%`
- Пузырь: `background: var(--color-user-bubble)` (фиолетовый градиент), `color: #fff`, `border-radius: 16px 16px 4px 16px`, `padding: 12px 18px`
- Время отправки: справа внизу под пузырём, `font-size: 11px`, `--color-text-muted`

**Сообщение ассистента:**
- Выравнивание влево, `max-width: 80%`
- Аватар бота (маленький, 32px, слева от пузыря)
- Пузырь: `background: var(--color-assistant-bubble)`, `border: 1px solid var(--color-border)`, `border-radius: 16px 16px 16px 4px`, `padding: 16px 20px`
- Контент: поддержка Markdown (заголовки, списки, жирный, курсив, код). Использовать `marked` + `DOMPurify` для рендера
- **Блок источников** под текстом ответа:
  - Заголовок: «Источники» с иконкой книги
  - Нумерованный список: «1. ГОСТ 34.602-89 "Техническое задание на создание АС"» — каждый кликабельный
  - `background: rgba(108, 92, 231, 0.08)`, `border-radius: 8px`, `padding: 12px`
  - Клик по источнику → открытие модального окна `SourceModal`
- **Кнопки действий** под сообщением ассистента (ряд иконок):
  - 👍 Like
  - 👎 Dislike
  - 📋 Копировать текст
  - 💬 «Оставить отзыв» — открытие `FeedbackModal`

**Индикатор «думает»:**
- Появляется при `isStreaming = true` и пустом ответе
- Пузырь ассистента с тремя анимированными точками (dot-pulse)

### 3d: Поле ввода (ChatInput)

- **Контейнер**: прижат к низу, `padding: 16px 24px`, `background: rgba(13, 16, 51, 0.7)`, `backdrop-filter: blur(10px)`
- **Textarea**: на всю ширину минус кнопка, `min-height: 44px`, `max-height: 200px`, авторасширение по контенту, `background: var(--color-bg-surface)`, `border: 1px solid var(--color-border)`, `border-radius: 12px`, `padding: 12px 50px 12px 16px`, `color: #fff`, placeholder «Задайте вопрос...»
- **Кнопка отправки**: круглая (`40px`), `background: var(--color-accent-gradient)`, иконка стрелки вверх, `position: absolute` справа внутри textarea-контейнера
  - Disabled: `opacity: 0.5` когда текст пустой или `isStreaming`
  - Active/hover: glow-эффект
- **Подсказка**: под textarea, мелким текстом `--color-text-muted`: «Shift + Enter для новой строки»
- Enter → отправка, Shift+Enter → перенос строки

## Модальное окно: Просмотр источника (SourceModal)

**Триггер**: клик по ссылке источника в ответе ассистента

### Визуал

- **Overlay**: `background: var(--color-overlay)`, на весь экран
- **Модальное окно**: `width: 600px`, `max-height: 80vh`, `background: var(--color-bg-elevated)`, `border: 1px solid var(--color-border)`, `border-radius: 16px`, `padding: 24px`
- Центрировано на экране

### Содержимое

1. **Header**: «Источник 1 из N» (с навигацией ←/→ между источниками), кнопка ✕ закрытия
2. **Название документа**: жирный, `font-size: 18px` — например «ГОСТ 34.602-89 "Техническое задание на создание автоматизированной системы"»
3. **Текст фрагмента**: `chunk_text` из sources, `font-size: 14px`, `line-height: 1.6`, `overflow-y: auto`
4. **Номер страницы** (если есть): «Страница 7» внизу справа, `--color-text-muted`

### Поведение

- ESC или клик по overlay → закрытие
- Стрелки ←/→ переключают между источниками данного сообщения
- Данные: массив `sources` из `MessageResponse`

  - `Ожидание` → серый бейдж
- **Действия** в каждой строке: иконка редактирования, иконка удаления (красная при hover)
- **Пагинация**: «← 1 2 3 ... 16 →» внизу таблицы, активная страница выделена `--color-accent-primary`

### API (Admin)

- `GET /api/admin/documents` — загрузка списка документов
- `POST /api/admin/documents` — загрузка файла (multipart/form-data)
- `DELETE /api/admin/documents/{id}` — удаление
- `POST /api/admin/documents/{id}/reindex` — переиндексация (stage 3)

> Примечание: эндпоинты admin ещё не описаны в openapi.yaml stage 1. На stage 1 AdminPage может быть заглушкой или использовать mock-данные.

## Общие UX-паттерны

### Загрузка данных
- При загрузке списков: skeleton-анимация (пульсирующие серые блоки)
- При отправке формы: кнопка disabled + спиннер внутри
- При загрузке страницы: полноэкранный спиннер на фоне `--color-bg-primary`

### Toast-уведомления
- Позиция: правый верхний угол
- Типы: success (зелёный), error (красный), info (фиолетовый)
- Автоисчезание через 4 секунды, кнопка ✕ для ручного закрытия

### Confirm-диалоги
- Для удаления чата, документа: модальное окно с вопросом, кнопки «Отмена» (ghost) и «Удалить» (красная)

### Пустые состояния
- Нет чатов: «Начните первый диалог!» + кнопка «Новый чат»
- Нет документов: иконка пустой папки + «Загрузите первый документ»
- Ошибка загрузки: «Не удалось загрузить данные» + кнопка «Повторить»

### Клавиатурные сочетания
- `Enter` — отправка сообщения
- `Shift+Enter` — новая строка
- `Escape` — закрытие модальных окон
- `Ctrl+N` / `Cmd+N` — новый чат (опционально)
