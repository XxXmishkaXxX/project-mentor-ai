# BE-113: Health check

**Приоритет**: Should have
**Этап**: 1
**US**: -
**Зависимости**: BE-003
**Статус**: не начата

## Описание

Расширить `GET /api/health`: проверка доступности PostgreSQL, Qdrant и Redis с агрегированным статусом ответа.

## Файлы

- `backend/src/app/main.py` — handler health с проверками зависимостей

## Реализация

- Попытка подключения/ ping к каждому сервису.
- Тело ответа JSON: `{"pg": "ok"|"error", "qdrant": "ok"|"error", "redis": "ok"|"error"}`.
- HTTP 200, если все сервисы ok; HTTP 503, если хотя бы один не доступен.

## Критерии готовности

- Ответ отражает реальное состояние каждого сервиса
- При падении одного из трёх — общий код 503
