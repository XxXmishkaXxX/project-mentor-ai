## Описание

Оркестрация RAG: эмбеддинг вопроса → поиск в Qdrant → сборка промпта → потоковый ответ LLM и выдача источников.

## Файлы

- `backend/src/app/rag/pipeline.py` — класс `RAGPipeline`, метод `ask`

## Реализация

- Класс `RAGPipeline` с зависимостями `embedder`, `retriever`, `llm`.
- `ask(question, chat_history=[]) -> AsyncGenerator` — события в формате SSE для чата.
- Логика: 1) эмбеддинг вопроса; 2) поиск top-k; 3) если все score ниже порога — событие «нет данных»; 4) system + контекстные чанки + история + вопрос; 5) стрим токенов от LLM; 6) событие с источниками.
- System prompt: отвечать только по контексту, ссылаться на источники, признавать отсутствие данных.

## Критерии готовности

- На вопрос вроде «Как составить ТЗ?» — ответ с указанием источников из базы
- На вопрос вне базы (например «Курс доллара?») — сообщение об отсутствии данных в корпусе
