## Описание

Прокручиваемая область со списком сообщений чата. Рендерит `MessageBubble` для каждого сообщения, автоматически прокручивается к последнему сообщению и во время стриминга. Показывает индикатор «думает» при ожидании ответа.

## Файлы

- `frontend/src/components/ChatWindow.vue` — контейнер сообщений с автоскроллом

## Визуальная спецификация

### Контейнер

- `flex: 1`, `overflow-y: auto`, `padding: 24px`
- Сообщения в flex-колонке с `gap: 16px`
- `scroll-behavior: smooth`
- Фон: прозрачный (виден анимированный космический фон через родительский элемент)

### Список сообщений

- Перебор `chatStore.messages` в хронологическом порядке (`created_at` ASC)
- Каждое сообщение → `<MessageBubble :message="msg" />`
- Ключ: `message.id`

### Индикатор «думает» (typing indicator)

Отображается когда `chatStore.isStreaming === true` и последнее сообщение ассистента ещё пустое:

- Пузырь в стиле ассистента (слева, с аватаром)
- Внутри: три точки с анимацией `pulse` с задержкой между точками:
  ```css
  .dot:nth-child(1) { animation-delay: 0s; }
  .dot:nth-child(2) { animation-delay: 0.2s; }
  .dot:nth-child(3) { animation-delay: 0.4s; }
  ```
- Каждая точка: `width: 8px`, `height: 8px`, `border-radius: 50%`, `background: var(--color-text-secondary)`

### Автоскролл

- **При новом сообщении**: `scrollTop = scrollHeight` (скролл вниз)
- **Во время стриминга**: каждый новый токен → скролл вниз
- **Умный скролл**: если пользователь вручную прокрутил вверх (расстояние от конца > 100px) — НЕ автоскроллить, чтобы не мешать чтению. Показать кнопку «↓» для ручного возврата вниз
- Реализация: `ref` на контейнер, `nextTick` + `scrollTo` / `scrollIntoView`

### Загрузка сообщений

- При первой загрузке чата: показать skeleton (3–4 серых пульсирующих блока разной ширины)
- При ошибке загрузки: «Не удалось загрузить сообщения» + кнопка «Повторить»

## API

- `chatStore.fetchMessages(chatId)` → `GET /api/chats/{chat_id}/messages?offset=0&limit=50`
- Ответ: `MessageListResponse { messages: MessageResponse[], total: number }`
- Пагинация: при прокрутке вверх до конца — подгрузить старые сообщения (`offset += limit`), вставить в начало списка

## Поведение

- При смене `currentChatId` → загрузить сообщения нового чата
- Во время стриминга: новые токены добавляются в `content` последнего сообщения ассистента
- После `done` event: `isStreaming = false`, сообщение финализировано
- Анимация появления новых сообщений: `fadeInUp` (opacity + translateY)

## Критерии готовности

- Сообщения отображаются в правильном хронологическом порядке
- Автоскролл работает при новых сообщениях и стриминге
- Умный скролл: не мешает чтению при ручной прокрутке вверх
- Индикатор «думает» с анимированными точками
- Skeleton при загрузке, ошибка с retry
