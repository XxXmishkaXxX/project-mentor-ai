# FE-112: SSE-стриминг

**Приоритет**: Must have
**Этап**: 1
**US**: US-02, US-03
**Зависимости**: BE-110
**Статус**: не начата

## Описание

Отправка сообщения с получением ответа по Server-Sent Events: инкрементальные токены в реактивное сообщение, событие с источниками, завершение стрима с `message_id`.

## Файлы

- `frontend/src/api/chat.js` — функция `sendMessage` с разбором SSE из `ReadableStream`

## Реализация

`fetch` POST на `/api/chats/{id}/messages` с `credentials: 'include'`. Читать `response.body` через `ReadableStream` и `TextDecoder`. Парсинг SSE: разбиение по строкам `event:` и `data:`. Обработка:

- `event:token` / соответствующий `data` — дописать текст к текущему ответу в store
- `event:sources` — сохранить массив источников в сообщение
- `event:done` — `isStreaming = false`, сохранить `message_id` если приходит

При статусе не 200 — проброс ошибки. Сетевые сбои — сообщение пользователю или в store. Синхронизация с `chatStore` (колбэки или передача writable ref).

## Критерии готовности

- Текст ответа появляется по мере прихода токенов
- Источники применяются после соответствующего события
- По `done` стрим корректно завершается
- Ошибки сети и HTTP обрабатываются
