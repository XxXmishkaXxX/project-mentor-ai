## Описание

Страница регистрации, визуально идентичная LoginPage: анимированный космический фон, центрированная полупрозрачная карточка. Поля: username, email, пароль, подтверждение пароля. Клиентская валидация + обработка серверных ошибок (дубликат email/username).

## Файлы

- `frontend/src/pages/RegisterPage.vue` — страница регистрации

## Визуальная спецификация

### Layout

Идентичен LoginPage: `min-height: 100vh`, `display: flex`, `align-items: center`, `justify-content: center`. CosmicBackground на заднем плане.

### Карточка формы

- Те же стили что LoginPage: `width: 420px`, `background: rgba(13, 16, 51, 0.85)`, `backdrop-filter: blur(20px)`, `border-radius: 16px`, `padding: 40px`
- Чуть больше в высоту из-за дополнительных полей

### Элементы (сверху вниз)

1. **Логотип** — идентичен LoginPage (SVG щит + «Проектный Ассистент»)
2. **Заголовок**: «Регистрация», подзаголовок: «Создайте аккаунт для начала работы»
3. **Поле «Имя пользователя»**: `<input type="text" placeholder="ivan_petrov">`, label «Имя пользователя». `margin-bottom: 16px`
4. **Поле «Email»**: `<input type="email" placeholder="you@example.com">`. `margin-bottom: 16px`
5. **Поле «Пароль»**: `<input type="password">`, иконка toggle видимости. `margin-bottom: 16px`
6. **Поле «Подтвердите пароль»**: `<input type="password">`. `margin-bottom: 24px`
7. **Кнопка «Зарегистрироваться»**: `width: 100%`, `height: 48px`, `background: var(--color-accent-green)`, аналог кнопки LoginPage. `margin-bottom: 24px`
8. **Ссылка**: «Уже есть аккаунт?» + `<router-link to="/login">Войти</router-link>`

## API-интеграция

### Отправка формы

```
POST /api/auth/register
Content-Type: application/json
Body: { "username": "...", "email": "...", "password": "..." }
```

Примечание: поле `confirm_password` проверяется только на клиенте, на сервер не отправляется.

### Обработка ответов

| Код | Действие |
|-----|----------|
| 201 | Регистрация успешна. Редирект на `/login`. Опционально: показать toast «Регистрация успешна! Войдите в аккаунт» |
| 409 | Показать текст из `response.detail` — «Пользователь с таким email или именем уже существует» |
| 422 | Показать ошибки валидации из `response.extra[]` под полями |
| Network error | Показать «Ошибка сети. Проверьте подключение» |

## Клиентская валидация

| Поле | Правило | Сообщение при ошибке |
|------|---------|----------------------|
| Username | Обязательно, 3–50 символов | «Имя должно быть от 3 до 50 символов» |
| Email | Обязательно, формат email | «Введите корректный email» |
| Пароль | Обязательно, минимум 8 символов | «Пароль должен быть не менее 8 символов» |
| Подтверждение | Совпадает с паролем | «Пароли не совпадают» |

- Валидация при blur и при submit
- Ошибки: красный текст под полем + `border-color: var(--color-accent-red)`
- Кнопка disabled пока есть ошибки валидации или пустые обязательные поля

## Поведение

- При монтировании: если `authStore.isAuthenticated` → `router.push('/chat')`
- Фокус на поле Username при загрузке
- Кнопка loading-состояние (спиннер) при отправке

## Критерии готовности

- Визуально идентична LoginPage по стилю карточки и фона
- Все 4 поля с валидацией на клиенте
- При 409 — понятное сообщение о дубликате
- При успехе — редирект на `/login`
- Загрузочное состояние кнопки при отправке
