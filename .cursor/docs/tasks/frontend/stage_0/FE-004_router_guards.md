## Описание

Маршрутизация приложения с разделением публичных и защищённых страниц. Глобальный `beforeEach` проверяет аутентификацию и роль администратора для `/admin`.

## Файлы

- `frontend/src/router/index.js` — определение маршрутов и navigation guards

## Реализация

Маршруты:

- `/login` — `LoginPage`, публичный
- `/register` — `RegisterPage`, публичный
- `/` — редирект на `/chat`
- `/chat` — `ChatPage`, только для авторизованных
- `/chat/:id` — `ChatPage`, только для авторизованных
- `/admin` — `AdminPage`, только `role === 'admin'`

Guards: в `beforeEach` читать Pinia auth store (после инициализации приложения). Неавторизованный доступ к защищённым маршрутам → `/login`. Пользователь без прав админа на `/admin` → `/chat`. Залогиненный на `/login` или `/register` — по необходимости редирект на `/chat` (опционально, зафиксировать в коде).

## Критерии готовности

- Маршруты открываются согласно таблице
- Guards блокируют неавторизованный доступ и не-админов на `/admin`
