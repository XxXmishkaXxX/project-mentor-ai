# FE-003: HTTP-клиент

**Приоритет**: Блокирующий
**Этап**: 0
**US**: -
**Зависимости**: FE-001
**Статус**: не начата

## Описание

Единая обёртка над HTTP: базовый URL, отправка cookie с каждым запросом, единообразная обработка ответов. При истечении сессии (401) пользователь перенаправляется на страницу входа.

## Файлы

- `frontend/src/api/client.js` — axios instance или обёртка над `fetch` с общими настройками и перехватчиками

## Реализация

Реализовать клиент с `credentials: 'include'` (для fetch) или `withCredentials: true` (для axios). Базовый URL: относительный `/api` (совместно с FE-002). Interceptor на ответы: при статусе 401 — редирект на `/login` (через `window.location` или `router.push`, избегая циклов на публичных маршрутах). Обработка сетевых ошибок по проектным правилам.

## Критерии готовности

- При 401 выполняется редирект на `/login`
- Cookie автоматически отправляются с запросами
- Остальные API-модули могут использовать этот клиент
