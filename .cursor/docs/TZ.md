# Техническое задание: ИИ-ассистент для проектной деятельности

## Описание проекта

Студенты, работая над проектами, часто сталкиваются не с техническими, а с методологическими трудностями:
как планировать задачи, вести документацию, распределять роли.

Данный проект направлен на разработку ИИ-ассистента, который станет для студентов "карманным" ментором.

В отличие от стандартных LLM, ассистент должен:

* давать ответы **только на основе проверенной базы знаний**
* использовать материалы: статьи, книги, ГОСТы, практики Agile, Scrum, DevOps и др.

---

## Стек технологий

| Компонент | Решение |
|-----------|---------|
| Язык программирования | Python 3.12 |
| Backend-фреймворк | Litestar (async) |
| Frontend | Vue 3 + Vite + Pinia + Vue Router |
| LLM | Qwen 3.6 API |
| Embedding-модель | Qwen Embedding API |
| Реляционная БД | PostgreSQL 16 |
| Векторная БД | Qdrant |
| Кеш / Сессии | Redis 7 |
| ORM / Миграции | SQLAlchemy 2.0 (async) + Alembic |
| RAG-пайплайн | Собственная реализация (без LangChain / LlamaIndex) |
| Авторизация | Сессии на Redis (cookie-based) |
| Стриминг ответов | SSE (Server-Sent Events) |
| Инфраструктура | Docker Compose (PostgreSQL, Qdrant, Redis) |

---

## Формат продукта

Веб-сайт (SPA). Пользователь открывает страницу в браузере, задаёт вопрос по проектной деятельности и получает ответ, основанный на подготовленной базе знаний. Ответы стримятся в реальном времени.

---

## Этапы работы

### 1. Сбор и структурирование базы знаний

* Подбор материалов по управлению проектами и инженерным практикам
* Очистка и подготовка данных
* Разделение на фрагменты (500-1000 токенов, overlap ~100 токенов)

### 2. Настройка векторной базы данных

* Развёртывание Qdrant в Docker
* Создание коллекции `knowledge_base`
* Генерация embeddings через Qwen Embedding API
* Загрузка и индексация базы знаний

### 3. Разработка RAG-пайплайна

* Клиент Qwen 3.6 API с поддержкой streaming
* Реализация цепочки:
  * embed вопроса пользователя
  * поиск top-k релевантных фрагментов в Qdrant
  * формирование промпта с контекстом
  * генерация и стриминг ответа через SSE

### 4. Разработка backend

* REST API на Litestar
* Авторизация через сессии (Redis)
* CRUD чатов и сообщений (PostgreSQL)
* Загрузка и индексация документов (admin)
* Сбор обратной связи

### 5. Разработка frontend

* SPA на Vue 3 с Vite
* Страницы: авторизация, чат, админ-панель
* SSE-стриминг ответов
* Отображение источников под ответом
* Адаптивная вёрстка

### 6. Тестирование и улучшение

* Проверка качества ответов на наборе 20+ тестовых вопросов
* Итеративное улучшение промптов и параметров retrieval
* Оценка релевантности, полноты, скорости ответов

---

## Уровни результата

### Минимальный уровень

* Сформирована и векторизована база знаний в Qdrant
* Реализован RAG-пайплайн: запрос -> поиск -> ответ
* Рабочий веб-интерфейс с авторизацией
* Ответы содержат ссылки на источники
* Корректная обработка отсутствия данных в базе

### Базовый уровень

* Полноценный чат с историей сообщений
* SSE-стриминг ответов
* Примеры вопросов на пустом чате
* Проверка на 20+ тестовых вопросах

### Оптимальный уровень

* Поддержка контекста диалога (история в промпте)
* Сбор обратной связи (полезно / не полезно + комментарий)
* Админ-панель: загрузка, удаление, переиндексация документов
* Адаптивная вёрстка для мобильных устройств

## Цель проекта

Получение практических навыков:

* разработки LLM-приложений с использованием RAG
* работы с векторными базами данных
* построения full-stack веб-приложений (Litestar + Vue 3)
* систематизации знаний в области:
  * инженерных практик
  * управления IT-проектами

---

## Требуемый результат

Готовый продукт в виде веб-сервиса.

Функциональность:

* консультирование студентов по вопросам ведения проектов
* ответы на вопросы, например:
  * "Как правильно составить ТЗ?"
  * "Какие этапы у Scrum?"
* использование загруженной базы знаний с отображением источников
* стриминг ответов в реальном времени
* история диалогов
* обратная связь от пользователей
* административное управление базой знаний

---

## Критерии оценки

1. Ассистент развёрнут и доступен для тестирования

2. Качество ответов:
   * проверено на наборе из 20+ тестовых вопросов
   * ответы релевантны и основаны на базе знаний
   * каждый содержательный ответ содержит ссылки на источники

3. Документация:
   * оформленная кодовая база проекта
   * подготовленная и описанная база знаний
   * документ архитектуры системы
